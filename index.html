Summary

I am a web developer with experience working on both front end and back end projects, including extensive exposure to MEAN stack JavaScript technologies and modern, responsive CSS design.

My main goal in my work is to collaborate closely with others, be they developers, non technical team members or clients, to choose the most appropriate solution to a given set of user needs and then to use that to create a product that is intuitive and powerful. This can mean quickly prototyping skeleton pages with Twitter Bootstrap then working incrementally with a designer to produce a fully bespoke final version or helping a client decide between a fully custom app hosted on AWS or a relatively simple WordPress site.

My recent personal development projects include a small app to allow a player of the game Android Netrunner (well, me, but it could be used by another player) to track their level of success with certain types of decks against others.

Skills

JavaScript

I've done extensive work with JavaScript, both in the browser and on the server with NodeJS and Express. Recently the bulk of my work has been with AngularJS, which I feel provides a large amount of flexibility and power without requiring excessive amounts of boilerplate to get started with a project. This ties in well with other parts of the JavaScript ecosystem, particularly partnered with npm, to create something with the minimal replication of work that has already been done, allowing me to get on with writing the things that are novel or unique to the current project.

Having worked on products that were required to run the same code on the server and in the browser, including older versions of IE and Safari to the same result I've gained a healthy respect for some of the particular rough edges that need to be avoided to make things work everywhere (particularly around events and concurrency).

I've also had some exposure to technologies like TypeScript and Angular 2.0 through courses and building toy personal projects, although I've not used either in production.

CSS

I have worked a lot with CSS, particularly with the LESS pre-processor, to produce attractive pages that provide users with the same experience, or as close as possible to it, on a wide range of devices and browsers. While working at Lighthouse Learning in particular, I was working on an application that had to provide the same on-brand experience on an interactive whiteboard and an iPhone. I've worked on projects that include CSS frameworks, particularly Twitter Bootstrap and others that build everything from scratch.

Project Management

This gave me a lot of opportunities to work directly with people from all parts of the company to ensure we were all working towards the same goals, as well as time to get better at estimating how long projects should take, taking into account how busy the various participants were and to improve my own time management.
